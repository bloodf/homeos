# HomeOS v1.2.0 — Selectable AI skills and complete AI integration docs

## Highlights

- Adds selectable `npx skills` installation records:
  - format: `source|agents|skills`
  - supports many records separated by semicolons
  - supports many target agents and many skills per record
  - supports `*` for all skills in a package
- Adds interactive checklist help when `whiptail` is available:
  - component checklist
  - AI skill package checklist
  - AI skill target-agent checklist
  - highlighted items show what the item does and what to think about before enabling it
- Adds Kimi/Gemini alias normalization for `npx skills`:
  - `kimi` → `kimi-cli`
  - `gemini` → `gemini-cli`
- Adds full AI integration documentation:
  - project repo links and descriptions
  - default target tools
  - npx skill source links and default skill selections
  - sanitized local MCP inventory for Pi, Claude Code, Codex, OpenCode, and Kimi
  - local skill/agent counts from `~/.agents`, Claude Code, Codex, OpenCode, and Pi
- Keeps MCP behavior unchanged:
  - HomeOS documents MCP inventory but does not rewrite global MCP configs
  - MCP/plugin directories remain per-tool
  - shared skills/agents remain separate from per-tool MCP/plugin state
