# HomeOS v1.0.0 — Universal installer release

Release date: 2026-05-04
Tag: `v1.0.0`

HomeOS v1.0.0 ships the production-ready universal shell installer as the primary install path. The old ISO/QEMU flow remains in the repository, but this release focuses on fast, repeatable installs on Debian, Ubuntu, and Fedora/RHEL-family systems.

## Highlights

- Adds `universal-installer/install.sh`, a single-file installer for Debian 12+, Ubuntu 22.04+, Fedora 38+, and RHEL-family systems.
- Supports interactive, unattended, minimal, dry-run, and uninstall modes.
- Installs the HomeOS core: base packages, admin user, Docker CE, Node.js/npm, pnpm, Bun, Tailscale, Caddy, Cockpit, CasaOS, app stacks, monitoring, Watchtower, AI CLIs, GitHub tools, firewall rules, SSH hardening, and backups depending on selected mode/config.
- Adds `/usr/local/bin/homeos` day-2 CLI with `status`, `doctor`, `logs`, `restart`, `backup`, `config`, `update`, `uninstall`, and `--version` commands.
- Adds config loading from `--config`, `/etc/homeos/homeos.conf`, `~/.config/homeos/homeos.conf`, or `./homeos.conf`.
- Adds preflight checks for disk, RAM, internet connectivity, and OS compatibility.
- Tracks install state under `/var/lib/homeos`.

## Security and reliability hardening

- Removed unsafe config `eval`; only strict `$VAR` / `${VAR}` environment expansion is allowed.
- Hardened random secret generation under `set -euo pipefail`.
- Generates random unattended admin passwords under `/var/lib/homeos/admin-password.txt`.
- Generates random Grafana admin passwords under `/var/lib/homeos/grafana-password.txt`.
- Fixed non-interactive uninstall flows and `--yes uninstall` / `uninstall --yes` argument ordering.
- Made uninstall clean up HomeOS-owned SSH, sudoers, Caddy, Docker daemon, and 45Drives artifacts while preserving system packages by default.
- Made systemd, Docker daemon, UFW/firewalld, and third-party installer failures degrade gracefully in containers/non-systemd environments.
- Fixed Debian/Ubuntu NodeSource setup, Fedora `wheel` group handling, GPG batch mode, UFW syntax, container RAM detection, and `homeos status` minimal-container fallback.

- Targeted uninstall parser tests: `UNINSTALL_PARSE_OK`, `UNINSTALL_FIRST_ARG_OK`

## Known limitations

- Docker app stacks require a working Docker daemon; basic Docker-in-Docker test containers only verify compose file generation and graceful daemon failure handling.
- Tailscale and some vendor install scripts depend on third-party package/script availability and may be slow or transiently unavailable in containers.
- Fedora 40 default repositories provide Node.js v20; the installer accepts that path for Fedora-family systems.
- CasaOS and several AI CLI installers rely on upstream vendor scripts; failures are non-fatal and reported as warnings.
- Full package-removal uninstall and advanced Grafana bind-address hardening are planned for the next development cycle.

## Install

```bash
curl -fsSL https://raw.githubusercontent.com/bloodf/homeos/main/universal-installer/install.sh | sudo bash
```

Minimal unattended example:

```bash
curl -fsSL https://raw.githubusercontent.com/bloodf/homeos/main/universal-installer/install.sh -o install.sh
sudo bash install.sh --unattended --mode minimal
```
