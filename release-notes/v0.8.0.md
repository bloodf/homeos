# HomeOS v0.8.0 — security, supply-chain, docs, and CI hardening

Release date: 2026-05-01

## Highlights

- Documented Docker socket and privileged-container residual risks across Cosmos,
  Portal, Dockge, Watchtower, Home Assistant, Jellyfin, and optional monitoring.
- Kept Cosmos on the HomeOS Docker socket audit shim rather than mounting the host
  Docker socket directly.
- Hardened secret prompts in the `ai-keys` installer so existing values are not
  partially printed and new values are entered silently.
- Replaced several direct downloaded-script execution paths with temp-file
  downloads, plus optional SHA256 support for `ai_clis_curl` entries.
- Replaced the optional Trivy installer apt-key flow with a scoped signed keyring.
- Added pinned Debian base ISO checksums in `build/debian-base-isos.sha256`; the
  downloader now verifies both the local manifest and upstream `SHA256SUMS`.
- Improved `make pin-tools` so all GitHub pins, including `hermes_agent_ref`, are
  refreshed deterministically and partial GitHub API failures refuse writes.
- Changed tag/manual ISO builds to verify committed GitHub pins instead of
  refreshing them during release builds.
- Added `make check-static` and wired it into the existing tag/manual-only ISO
  workflow without adding branch push or pull-request triggers.
- Documented release artifact verification using generated `.iso.sha256` files.

## CI and artifact policy

The ISO workflow remains manual/tag-only:

- `workflow_dispatch` builds amd64 and arm64 artifacts.
- `v*` tags build amd64 and arm64 artifacts and attach ISOs plus `.sha256` files
  to the GitHub release.
- Branch pushes and pull requests do not trigger ISO builds.

Verify downloaded release artifacts with:

```bash
sha256sum -c homeos-debian-13.4-amd64.iso.sha256
sha256sum -c homeos-debian-13.4-arm64.iso.sha256
```

## Accepted residual risks

- Dockge, Watchtower, and Homepage still require host Docker socket access for
  their intended features. Treat access to those UIs as host-root equivalent.
- Portal ttyd web terminals mount admin home and secrets read-only, but are now
  loopback-bound behind Caddy instead of host-network exposed on the tailnet.
- Cosmos still runs privileged because it is an optional container manager, but
  its Docker API traffic goes through the audit shim.
- Home Assistant and optional Scrutiny monitoring retain privileged/device access
  for integrations and SMART data.
- Several application containers intentionally use moving image tags (`latest`,
  `stable`, or `main`). Critical services are excluded from automatic Watchtower
  updates where configured.
- Some upstream installers do not publish stable checksums; HomeOS downloads them
  over TLS, documents the risk, and executes from reviewed URLs.
- The builder image uses `debian:trixie-slim` by tag and installs the explicit
  apt tool list from current Debian trixie metadata.

## Validation

No QEMU, VM boot, or ISO build was run for this worker hardening pass. Static
validation covered shell syntax, YAML parsing, workflow parsing, forbidden marker
string checks outside `ROADMAP-TO-0.9.md`, and Ansible syntax check when available.
