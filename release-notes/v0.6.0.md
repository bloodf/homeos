# HomeOS v0.6.0 — Audit replay and sidecar payloads

v0.6.0 adds replayable audit sidecars for mutating `homeos` CLI actions.

## Highlights

- Gated mutating CLI commands require `sudo` and fail closed if audit sidecar or JSONL writing fails.
- The same gated action now writes a root-only replay sidecar under `/var/lib/homeos/audit-replay/` with original argv, intent/diff summary, replay-safe environment metadata, user/timestamp, diff hash, and redaction policy.
- `homeos audit show <id_or_hash>` prints the public audit entry; root also sees sidecar JSON, while non-root gets a clear root-only refusal.
- `sudo homeos audit replay <id_or_hash>` resolves a line-number ID or unique diff hash and re-runs the stored argv through the AI gate.
- Replay writes a fresh audit line with `cmd: "audit:replay:<orig_cmd>"`.
- Duplicate diff hashes are safe: the first sidecar uses `<hash>.json`, later duplicates get unique suffixed `sidecar_id` values and ambiguous bare-hash replay asks for a line-number ID.
- `homeos-audit-prune.timer` prunes replay sidecars after 90 days. Public JSONL rotation remains 10 years.

## Secret handling

Secret values are not written to public JSONL. For example, `sudo homeos config secrets set KEY=VALUE` records `cmd=secrets:set:KEY`, verdict, choice, diff hash, and sidecar id publicly; the replayable argv containing the value is stored only in the root-only sidecar.

## Validation

Worker validation for this implementation used static shell checks, YAML parsing, and a temp-directory audit show/replay/prune harness. No QEMU, VM boot, ISO build, commit, tag, or push was run.
