# HomeOS v1.1.0 — App hosting, local domains, Pi, and dashboards

## Highlights

- Adds local wildcard domains with `dnsmasq`:
  - default root: `homeos.home.arpa`
  - wildcard: `*.homeos.home.arpa -> HomeOS server IP`
  - configure router/client DNS to use HomeOS for local resolution
- Adds Caddy route management through the `homeos` CLI:
  - `homeos domain add app 3000`
  - `homeos domain list`
  - `homeos domain remove app`
- Adds Coolify installation using the official Coolify installer.
- Adds Pi coding agent installation from `pi.dev`/npm:
  - installs `@mariozechner/pi-coding-agent`
  - installs the maintainer workstation Pi package set by default (`context-mode`, `pi-subagents`, `pi-lens`, `taskplane`, `forgeflow-dev`, `pi-processes`, memory, Mermaid, and more)
- Adds an isolated AI project library with all requested helper repositories:
  - includes oh-my-claudecode, claude-mem, A11Y.md, code-review-graph, hindsight, taste-skill, portless, Matt Pocock skills, cinsights, claude-context, ClawTeam, heretic, OpenViking, impeccable, agency-agents, oh-my-openagent, shannon, hive, superpowers, and AgentTower
  - supports per-project tool targeting via `AI_PROJECT_TARGETS`
  - shares skills/agents while keeping MCP/plugin directories per-tool and leaving global MCP config untouched
- Improves Grafana monitoring:
  - adds node-exporter to the monitoring stack
  - provisions the Prometheus datasource automatically
  - provisions a `HomeOS Server Overview` dashboard for CPU, RAM, disk, and network throughput

## Validation

- `make check`
- `make smoke`
- Smoke coverage includes config-injection safety, uninstall parser safety, Grafana password/bind/dashboard generation, local wildcard DNS config generation, and AI project isolation/linking in manifest-only mode.
