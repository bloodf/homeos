# HomeOS v0.9.0 — release-candidate polish

Release date: 2026-05-02

## Highlights

- Polished documentation after the v0.5-v0.8 release train so user-facing docs describe current behavior instead of old future-work assumptions.
- Confirmed optional-installer docs, security notes, architecture notes, and release notes align with the current codebase.
- Added the v1.0 final validation checklist/runbook that keeps full QEMU and final ISO validation explicitly deferred until the v1.0 gate.
- Re-ran static release-candidate validation without local ISO builds or VM boots.

## CI and artifact policy

The ISO workflow remains tag/manual-only:

- `workflow_dispatch` builds amd64 and arm64 artifacts.
- `v*` tags build amd64 and arm64 artifacts and attach ISOs plus `.sha256` files to the GitHub release.
- Branch pushes and pull requests do not trigger ISO builds.

Verify downloaded release artifacts with:

```bash
sha256sum -c homeos-debian-13.4-amd64.iso.sha256
sha256sum -c homeos-debian-13.4-arm64.iso.sha256
```

## Known limitations before v1.0

- Full fresh-install QEMU validation is intentionally deferred to v1.0.
- Real hardware bootstrap completion, service health after reboot, and secure-mode lockout resistance must be proven during the v1.0 visible validation run.
- Raspberry Pi images remain outside the v1.0 ISO validation scope; v1.0 targets the current Debian ISO flow.

## Validation

No QEMU, VM boot, local ISO build, or final ISO smoke was run for this RC polish pass. Validation covered shell syntax, Python syntax, YAML parsing, forbidden marker checks, committed GitHub tool pin verification, release-note presence, docs consistency review, audit replay static harnessing, Cosmos shim static harnessing, and Ansible syntax check when available.
