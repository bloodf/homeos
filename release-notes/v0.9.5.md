# HomeOS v0.9.5 — multi-model review hardening

Release date: 2026-05-02

## Highlights

- Addressed GLM 5.1, Kimi K2P6, and MiniMax M2.7 review findings against the latest HomeOS implementation.
- Pinned GitHub Actions to immutable commit SHAs and scoped `contents: write` permission to the release-attachment job only.
- Hardened `build/refresh-pins.sh` with curl retries/timeouts and JSON parsing for GitHub commit responses.
- Tightened Cosmos Docker shim socket permissions from `0660` to `0600`.
- Replaced audit logrotate `copytruncate` with `create 0644 root root` to avoid truncation races.
- Hardened NAS label handling, NAS YAML updates, NFS/export ownership, and exfat/ntfs mount ownership.
- Updated Vaultwarden and Homepage portal configuration to use tailnet-aware hostnames and tighter allowed hosts.
- Hardened offsite backup environment writes to update keys in place, reject embedded newlines, and validate remote names/paths.
- Cleaned stale operational docs and added `REVIEW-FIX-REPORT.md` summarizing review findings, actions taken, validation, and deferred candidates.

## CI and artifact policy

The ISO workflow remains tag/manual-only:

- `workflow_dispatch` builds amd64 and arm64 artifacts.
- `v*` tags build amd64 and arm64 artifacts and attach ISOs plus `.sha256` files to the GitHub release.
- Branch pushes and pull requests do not trigger ISO builds.

Verify downloaded release artifacts with:

```bash
sha256sum -c homeos-debian-13.4-amd64.iso.sha256
sha256sum -c homeos-debian-13.4-arm64.iso.sha256
```

## Validation

No QEMU, VM boot, or local ISO build was run for this patch release. Validation covered:

- `make check-static`
- committed GitHub tool pin verification with `build/refresh-pins.sh --check`
- shell syntax checks for touched shell scripts
- Cosmos shim Python syntax check
- targeted audit/Cosmos harness for `audit show`, `audit replay`, and `audit cosmos-events`
- workflow YAML parse spot-check
- `git diff --check`

`ansible-playbook --syntax-check bootstrap/install.yml` was skipped locally because `ansible-playbook` is unavailable.
