# HomeOS v0.5.0

## Highlights

- Replaces Cosmos v0.4 log-tail auditing with a Docker API Unix-socket shim.
- Cosmos now mounts `/var/run/cosmos-docker.sock` as `/var/run/docker.sock`; the real Docker socket is not mounted directly.
- Mutating Docker API methods from Cosmos (`POST`, `PUT`, `PATCH`, `DELETE`) are mirrored into `/var/log/homeos-audit.jsonl` as `BYPASS` audit entries.
- Read-only Docker API calls proxy without audit entries, and request bodies are never logged.
- `homeos status` and `homeos config cosmos status` now surface the shim service state.

## Verification

- Python shim bytecode compilation.
- Bash syntax check for changed shell scripts.
- YAML parse for changed Ansible files.
- Fake Unix-socket smoke test proving `GET` proxies without audit and `POST` proxies with a `BYPASS` audit entry.
- Light QEMU scope remains: ISO boot + unattended installer path only, not a full bootstrap/service run.
