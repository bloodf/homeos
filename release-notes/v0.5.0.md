# HomeOS v0.5.0

## Highlights

- Replaces Cosmos v0.4 log-tail auditing with a Docker API Unix-socket shim.
- Cosmos now mounts `/var/run/cosmos-docker.sock` as `/var/run/docker.sock`;
  the real Docker socket is not mounted directly.
- Mutating Docker API methods from Cosmos (`POST`, `PUT`, `DELETE`) against
  containers, images, networks, and volumes are mirrored into
  `/var/log/homeos-audit.jsonl` as `cosmos:<verb>:<resource>` / `BYPASS` audit
  entries.
- Read-only Docker API calls proxy without audit entries, and request bodies are
  never logged; the audit line keeps only hash/size/truncation metadata.
- `homeos status` and `homeos config cosmos status` now surface the shim service state.

## Verification

- Python shim bytecode compilation.
- Bash syntax check for changed shell scripts.
- YAML parse for changed Ansible files.
- Fake Unix-socket smoke test proving `GET` proxies without audit and `POST`
  proxies with a `cosmos:post:containers` / `BYPASS` audit entry.
- No QEMU/VM boot validation is required for this worker milestone; final VM
  validation is deferred by `ROADMAP-TO-0.9.md`.
